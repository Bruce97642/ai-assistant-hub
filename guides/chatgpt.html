第二章　ChatGPT 詳解

2.1 什麼是 ChatGPT

ChatGPT 是 OpenAI 於 2022 年 11 月推出的 AI 對話服務，底層是「大型語言模型（LLM, Large Language Model）」。它透過閱讀海量文字資料學會語言規律，能理解你的問題並產生自然流暢的回應。截至 2026 年，ChatGPT 已是全球使用人數最多的 AI 應用之一，功能也從純文字對話擴展到圖像生成、語音互動、檔案分析、深度研究（Deep Research）與自動化代理。

2.2 運作原理（給非技術人員的版本）

2.2.1 三個必懂關鍵字

  • Token（詞元）：模型處理文字的最小單位。中文約 1 個字 ≈ 1 個以上的 token。模型的「記憶量」與「計費」都以 token 計算。
  • 上下文視窗（Context Window）：模型一次能「看見」的對話總量。超過上限，最早的內容會被遺忘，這解釋了為什麼長對話後 AI 會忘記前面說過的事。
  • 幻覺（Hallucination）：模型是在「預測最合理的下一段文字」，而不是查資料庫，因此可能自信地說出錯誤內容。重要資訊務必要求它附上來源，或自行查證。

2.2.2 一次對話發生了什麼事（流程拆解）

  • 你輸入文字（提示詞，Prompt），系統將它切成 token。
  • 模型根據「這次輸入 + 之前的對話 + 系統設定」計算最可能的回應。
  • 需要時，模型會呼叫工具：上網搜尋、執行程式、產生圖片、讀取你上傳的檔案。
  • 模型逐字生成回應並回傳給你；整段過程沒有「查詢標準答案」這回事。

💡 教學提示｜課堂比喻

可以把 LLM 比喻成「讀過整個圖書館、記憶力驚人但偶爾會腦補的博學同事」：它反應快、什麼都能聊，但關鍵數字與事實要幫它核對。

2.3 模型與方案（2026 年中現況）

  • GPT-5.5 系列（現行主力）：GPT-5.5 Instant 為預設模型，主打快速回應；GPT-5.5 Thinking 具深度推理能力，適合複雜分析；GPT-5.5 Pro 提供最高階效能，面向專業研究與高難度任務。
  • Codex 專用模型（gpt-5.x-codex 系列）：針對軟體工程最佳化，供 Codex 產品線使用。
  • 自動路由（Auto）：系統會依問題難度自動在快速與深度推理模型間切換，新手直接用預設即可。

第三章　ChatGPT 入門操作（逐步教學）

3.1 註冊與登入

  • 開啟瀏覽器，前往 chatgpt.com（手機請至 App Store / Google Play 搜尋「ChatGPT」，認明 OpenAI 官方開發者）。
  • 點選「Sign up（註冊）」，可用 Email，或直接以 Google／Microsoft／Apple 帳號快速註冊。
  • 完成信箱驗證與基本資料（姓名、生日）。
  • 登入後即可免費開始使用；升級付費方案可之後再說。

⚠ 注意｜常見卡關點

學員最常見的問題是：下載到山寨 App、忘記驗證信箱、公司網路擋住網站。課前請學員先完成註冊，可省下 20 分鐘課堂時間。

3.2 介面導覽（5 個必認識的區域）

  • 輸入框：畫面下方，打字的地方；旁邊的「＋」可上傳檔案、開啟工具（深度研究、圖像生成等）。
  • 模型選擇器：畫面上方，切換不同模型（新手用預設 Auto 即可）。
  • 左側邊欄：歷史對話清單、新對話按鈕、GPTs、專案（Projects）、Codex 入口。
  • 回應區工具：每則回應下方可複製、重新生成、朗讀、給予好壞評價。
  • 設定（Settings）：個人化、記憶管理、資料控制（是否允許用你的對話訓練模型）、語言。

3.3 第一次對話：任務拆解示範

課堂示範任務：「請 ChatGPT 幫我寫一封請假 Email」。以下是把一個模糊需求變成好結果的完整步驟：

  • 先講清楚身分與情境：「我是行政人員，明天要請病假一天，主管是張經理。」
  • 說明任務與要求：「請幫我寫一封語氣禮貌、150 字內的請假 Email，附上職務代理安排。」
  • 檢視結果並追問修改：「語氣再正式一點」「幫我加上聯絡電話的欄位」。
  • 滿意後複製使用。提醒學員：寄出前務必自己讀過一遍。

3.4 提示詞（Prompt）基礎：CRTF 四要素框架

寫好提示詞是本課程最重要的技能。建議用四個要素檢查每一次的輸入：

要素英文說明範例片段
角色Character / Role告訴 AI 它是誰「你是一位資深行銷顧問」
任務Request / Task要它做什麼，愈具體愈好「幫我寫 3 個 Instagram 貼文標題」
脈絡Context背景、對象、限制條件「產品是手工果醬，客群是 30–45 歲家庭主婦」
格式Format希望的輸出形式「用表格呈現，每個標題附 15 字說明」

完整範例（課堂可直接展示前後對比）

弱提示詞：「幫我寫貼文」→ 結果泛泛而談。

你是一位擅長社群行銷的文案專家。（角色）

請幫我寫 3 個 Instagram 貼文標題與內文。（任務）

產品是手工果醬，主打無添加，客群為 30–45 歲注重健康的家庭；

品牌語氣溫暖、不浮誇。（脈絡）

請用表格輸出：欄位為「標題、內文（80字內）、建議 hashtag」。（格式）

三個立即見效的進階技巧

  • 給範例（Few-shot）：「參考以下我寫過的兩篇貼文風格……」，AI 會模仿你的風格。
  • 要求分步思考：「請先列出大綱給我確認，再撰寫全文」，避免一次產出偏離需求的長文。
  • 迭代而非重來：結果不滿意時，具體指出要改哪裡，比重新問一次更有效。

💡 教學提示｜課堂練習（10 分鐘）

請學員各自用 CRTF 框架改寫「幫我規劃旅遊」這句弱提示詞，兩人一組互相比較 AI 的輸出差異，最後抽 2–3 組上台分享。

第四章　ChatGPT 應用方式（場景實戰）

本章每一節皆為獨立教學模組，包含：適用情境 → 操作步驟 → 示範提示詞 → 練習題，講師可依課程主題挑選組合。

4.1 文書寫作與改寫

適用：Email、公告、報告、簡報講稿、社群文案。

  • 用 CRTF 框架下第一版指令（角色＋任務＋脈絡＋格式）。
  • 先要「大綱」或「三個版本的開頭」，選定方向再展開全文。
  • 針對段落逐一修改：「第二段太長，濃縮成三句」。
  • 最後要求「用挑剔主管的角度檢查這篇文章的問題」，做自我審稿。

示範提示詞：「你是企業內部溝通專家。請將以下公告改寫得更精簡友善，保留所有日期與規定，控制在 200 字內，並列出你修改了哪些地方：（貼上原文）」

4.2 長文件摘要與資料整理

適用：會議記錄、合約、研究報告、多份 PDF 的比較。

  • 點輸入框旁的「＋」上傳檔案（PDF／Word／Excel 皆可，可一次多份）。
  • 指定摘要的「視角與用途」：「用要向老闆匯報的角度，整理三大重點與待決事項」。
  • 要求結構化輸出：表格、條列、或「風險／機會」兩欄對照。
  • 針對細節追問：「第 12 頁提到的違約條款，白話解釋給我聽」。

💡 教學提示｜重要觀念

摘要品質取決於「你告訴它這份摘要要給誰看、拿來做什麼」。同一份文件，給法務看與給業務看的摘要應該完全不同。

4.3 翻譯與跨語言溝通

  • 不要只說「翻譯」，要說明文體：「翻成商務英文 Email，語氣專業但友善」。
  • 要求雙重輸出：「附上逐句對照，並標註你調整語氣的地方」。
  • 反向驗證：請它把譯文再翻回中文，檢查意思是否走樣。

4.4 學習輔助（家教模式）

  • 設定教學角色：「你是有耐心的高中數學家教，用蘇格拉底式提問引導我，不要直接給答案」。
  • 讓 AI 出題→學生作答→AI 講評，形成練習循環。
  • 要求「用三種不同比喻解釋這個概念」，找到最適合自己的理解方式。
  • 結束時要求產出「本次學習重點卡＋5 題複習題」。

4.5 資料分析與 Excel 協作

  • 上傳 Excel／CSV 檔，先問：「這份資料有哪些欄位？品質有什麼問題？」
  • 下分析指令：「依月份統計各分店營收，找出成長最快與衰退最多的分店」。
  • 要求視覺化：「畫出趨勢折線圖」，並請它解讀圖表意義。
  • 請它產出可直接貼回 Excel 的公式或整理後的新檔案。

⚠ 注意｜資安提醒

上傳公司資料前，先確認公司政策；可先去識別化（刪除姓名、客戶名稱），並到「設定 → 資料控制」關閉「用我的內容改善模型」。

4.6 深度研究（Deep Research）

適用：市場調查、競品分析、政策研析、論文文獻初探。

  • 在輸入框的工具選單中選擇「Deep Research／深度研究」。
  • 像交辦研究員一樣下指令：說明研究問題、範圍、時間區間、希望的報告結構。
  • AI 可能反問釐清問題，認真回答可大幅提升品質。
  • 等待數分鐘後取得附引用來源的完整報告，逐一抽查關鍵引用是否屬實。

4.7 課堂綜合練習題庫

  • 【初級】用 CRTF 框架，請 AI 幫你寫一封活動邀請函，並迭代修改兩次。
  • 【中級】上傳一份 3 頁以上的 PDF，產出「給主管的 1 頁摘要」與「給同事的執行清單」兩個版本。
  • 【中級】上傳範例銷售資料，完成一次「統計→圖表→洞察→建議」的完整分析。
  • 【進階】用 Deep Research 完成一份競品分析，並驗證其中三個引用來源。

第五章　Codex 詳解

5.1 什麼是 Codex

Codex 是 OpenAI 的 AI 程式代理（AI Coding Agent）。它與 ChatGPT 最大的差別在於：ChatGPT「回答」你的程式問題，而 Codex 會「動手」完成整個開發任務——讀懂你的專案、修改檔案、執行測試、修 Bug，甚至直接開出 Pull Request 供你審查。

名詞演變（授課時務必先釐清）

  • 2021 年的 Codex：一個把自然語言轉成程式碼的「模型」，是 GitHub Copilot 早期的基礎，後來退役。
  • 2025 年起的 Codex：重新以「代理產品」之姿登場——2025 年 4 月推出開源的 Codex CLI，5 月推出雲端代理，之後陸續加入 IDE 擴充、macOS 桌面 App 與手機支援。現在講 Codex，指的是這一整套產品家族。

5.2 Codex 產品家族

介面在哪裡使用適合情境
Codex CLI終端機（macOS / Windows / Linux）本機開發主力；開源、以 Rust 打造，可讀寫並執行所選目錄的程式碼
IDE 擴充VS Code 等編輯器邊寫程式邊協作，即時預覽變更
Codex App（桌面）macOS 應用程式同時管理多個代理並行工作、內建 worktrees 隔離變更
Codex Cloud（雲端）ChatGPT 側邊欄 / 網頁把任務丟到雲端沙箱並行執行，完成後回報並可發 PR
行動裝置ChatGPT 手機 App外出時監控、引導、批准長時間執行的任務

5.3 五個核心概念

  • 代理迴圈（Agent Loop）：Codex 的工作方式是「思考 → 呼叫工具（讀檔、改檔、跑指令）→ 看結果 → 再思考」的循環，直到任務完成才回報。這是它與一問一答式聊天最根本的差異。
  • 沙箱（Sandbox）：預設情況下，Codex 只能在你指定的資料夾內讀寫，需要網路或更高權限時會先徵求同意，這是它的安全底線。
  • 審批模式（Approval Modes）：你可以決定 Codex 的自主程度——從「每個動作都要我批准」到「放手去做」。教學上建議新手從保守模式開始。
  • AGENTS.md：放在專案根目錄的說明檔，等於「給 AI 看的員工手冊」，寫入專案慣例、測試指令、程式風格，Codex 每次都會先讀它。
  • Skills 與 Automations：Skills 是可重複使用的技能包（指令＋資源＋腳本），讓 Codex 依團隊標準完成任務；Automations 則讓 Codex 依排程主動工作，例如每天自動分類 Issue、監控 CI 失敗。

5.4 如何取得 Codex

Codex 內含於 ChatGPT 的付費方案（Plus、Pro、Business、Edu、Enterprise；OpenAI 曾限時開放 Free/Go 體驗），也可改用 API 金鑰按用量計費。同一個 ChatGPT 帳號可在 CLI、IDE、App、雲端之間共用登入與使用額度。

💡 教學提示｜非工程師也能學嗎？

可以。Codex 正從「寫程式的代理」演變為「用程式幫你完成電腦工作的代理」，例如整理檔案、彙整資訊、產出文件。不過本教材第六、七章仍以開發情境為主軸，非技術班級可只教 5.1–5.3 的觀念與 7.5 的輕量應用。

第六章　Codex 入門操作（逐步教學）

6.1 前置準備

  • 一個 ChatGPT 付費帳號（或 OpenAI API 金鑰）。
  • 已安裝 Node.js 與 npm（用 npm 安裝 CLI 最方便；macOS 亦可用 Homebrew）。
  • Windows 使用者：可在 PowerShell 原生執行（含 Windows 沙箱），需要 Linux 環境時建議安裝 WSL2。
  • 一個練習用的程式專案資料夾（課堂可提供小型範例專案，避免學員用公司正式程式碼練習）。

6.2 安裝與登入 Codex CLI

  • 開啟終端機，執行安裝指令：

npm install -g @openai/codex

# macOS 也可以：brew install codex

  • 進入你的專案資料夾：

cd 你的專案路徑

  • 輸入 codex 啟動。首次執行會要求登入：選擇以 ChatGPT 帳號登入（推薦），瀏覽器會自動開啟授權頁面；或改用 API 金鑰。
  • 登入成功後會進入互動介面，看到輸入提示即代表安裝完成。
  • 日後升級：重新執行 npm install -g @openai/codex 即可更新到最新版。

6.3 第一個任務：讓 Codex 認識專案

  • 在 Codex 中輸入第一個問題（純閱讀、零風險）：

請解釋這個專案的整體架構，以及主要進入點在哪個檔案。

  • 觀察 Codex 如何自己讀取檔案、彙整結論——這就是代理迴圈的實際樣貌。
  • 接著給一個小型修改任務：

請在 README.md 加上一節「如何在本機啟動」的說明，

內容依照專案實際的啟動方式撰寫。

  • Codex 會顯示變更差異（diff）並依審批模式徵求同意；確認後變更才會寫入。
  • 用 git diff 或編輯器複查它改了什麼——「AI 動手、人類把關」是全程不變的原則。

6.4 審批模式與常用指令

自主程度行為建議對象
唯讀 / 建議只讀取與提出建議，不動任何檔案第一次上課、檢視陌生專案
自動編輯（預設區間）可在工作目錄內編輯檔案，執行指令前需批准日常開發（推薦起點）
完全自主（full access）可自由執行指令與存取網路熟練者、拋棄式沙箱環境；課堂上不建議示範

必教的斜線指令與操作

  • /model：切換模型（例如在快速與深度推理的 codex 模型間切換）。
  • /review：啟動程式碼審查——可審查未提交的變更、與主分支的差異、或指定的 commit。
  • /personality：切換代理的溝通風格（簡潔務實 vs. 詳細友善）。
  • codex &quot;單句任務&quot;：非互動模式，適合快速問答與寫進自動化腳本。
  • Ctrl+G：把長提示詞丟到你慣用的文字編輯器撰寫。

6.5 撰寫 AGENTS.md（決定 Codex 好不好用的關鍵）

  • 在專案根目錄建立 AGENTS.md 檔案。
  • 寫入三類資訊：專案簡介與架構、開發規範（語言、風格、命名）、常用指令（如何跑測試、如何啟動）。
  • 個人層級的偏好可另外寫在 ~/.codex/AGENTS.md，Codex 會由上而下合併多層設定。
  • 每次 Codex 行為不符期待時，回頭把規則補進 AGENTS.md——它會愈用愈順手。

第七章　Codex 應用方式（實戰流程）

7.1 修復 Bug 的標準流程

  • 描述症狀而非猜測原因：「使用者回報結帳時金額計算錯誤，重現步驟是……請先找出原因再修復」。
  • 要求先報告診斷：「先告訴我你認為的根本原因與修復計畫，經我同意再動手」。
  • 同意後讓它修復，並要求「同時新增一個能重現此 Bug 的測試案例」。
  • 檢視 diff、跑完整測試、確認後才提交。

7.2 開發新功能

  • 把需求寫成明確規格（輸入、輸出、邊界條件、不要做什麼）。
  • 大功能先要「實作計畫」：拆成哪些步驟、會動到哪些檔案。
  • 分批執行、分批審查，一次只合併一個可驗證的小段落。
  • 完成後用 /review 讓另一個 Codex 代理審查程式碼，再由人類做最終把關。

7.3 程式碼審查與重構

  • 審查未提交變更：提交前輸入 /review，選「未提交的變更」，取得高優先度的問題清單。
  • 客製審查重點：可下自訂指令，例如「專注檢查資安漏洞與效能問題」。
  • 遺留程式碼重構：先要求「只解釋、不修改」建立理解，再小步重構並以測試護航。

7.4 雲端代理與並行工作

  • 在 ChatGPT 側邊欄進入 Codex（或使用 Codex App / codex cloud），連結你的 GitHub 儲存庫。
  • 把多個「範圍明確的小任務」同時派給多個代理並行處理，每個任務都在獨立的雲端沙箱執行。
  • 任務完成後逐一審查結果，滿意的讓它直接建立 Pull Request。
  • 進階：設定 Automations，讓 Codex 依排程自動處理例行工作（Issue 分類、CI 失敗摘要、每日進度簡報）。

💡 教學提示｜派工心法

官方與社群的一致經驗：把任務切小、範圍講清楚，多個小任務並行的成果，遠勝一個模糊的大任務。這也是課堂上最值得反覆強調的工作習慣。

7.5 非工程場景的輕量應用

  • 整理資料夾：「把這個資料夾裡的發票 PDF 依年月重新命名並歸檔」。
  • 批次處理：「把這 50 張圖片全部轉成 1080p 的 JPG」。
  • 產出文件：「讀完這個專案後，幫我寫一份給新同事的上手指南」。
  • 搭配 Skills：把公司常用流程（如產出週報格式）做成技能包，全團隊共用。

第八章　進階提示詞工程（ChatGPT 與 Codex 通用）

8.1 六個進階原則

  • 先計畫後執行：複雜任務一律先要大綱／計畫，確認後再產出，成本最低的品質保險。
  • 明確的驗收標準：告訴 AI「怎樣算完成」——字數、格式、測試通過、涵蓋哪些要點。
  • 負面清單：明說不要什麼：「不要用條列式」「不要修改設定檔」。
  • 提供範例與素材：好範例勝過千言描述；給 Codex 附上截圖或設計稿效果更佳。
  • 自我檢查：產出後要求「列出這份成果最可能出錯的三個地方」。
  • 拆解大任務：超過一個段落講不清楚的需求，就拆成多輪對話或多個代理任務。

8.2 兩者提示詞的差異

面向ChatGPT（對話助理）Codex（執行代理）

9.1 模組對照表

9.2 授課通用原則

  • 每 20 分鐘講述搭配一次動手操作；AI 課程「看會」與「做會」差距極大。
  • 所有示範先在課前重跑一次——模型更新快，介面與輸出可能與講義不同，請以「原則不變、畫面會變」的心態教學。
  • 準備去識別化的範例檔案（假資料的 Excel、範例專案），嚴禁學員上傳真實客戶資料練習。
  • 每堂課保留 10 分鐘「翻車展示」：故意示範 AI 幻覺或失敗案例，建立正確的信任程度。

9.3 評量方式建議

  • 實作評量：給定情境，評分學員的提示詞品質（是否含角色、任務、脈絡、格式、驗收標準）。
  • 成果評量：繳交「原始需求 → 提示詞 → AI 輸出 → 人工修正」的完整過程紀錄，而非只看最終成品。
  • Codex 班：以 Git 紀錄評量——是否小步提交、是否有測試、是否留下 AGENTS.md。

第十章　常見問題與風險守則

10.1 常見問題 FAQ

Q1：ChatGPT 的回答可以直接相信嗎？

不行。它可能產生幻覺。原則：創意類內容放心用、事實類內容必查證、數字與法條一律核對原始來源。

Q2：免費版夠用嗎？

學習與輕度使用夠用。當你頻繁碰到額度上限、需要深度研究或 Codex 時，再依需求升級。

Q3：公司資料能上傳嗎？

先確認公司政策。個人帳號建議關閉「以內容改善模型」；企業建議採用 Business／Enterprise 方案，資料保護與管理權限較完整。

Q4：Codex 會不會弄壞我的程式？

防線有四層：沙箱限制範圍、審批模式控制權限、Git 版本控制可隨時還原、人工審查 diff。守住這四層，風險非常可控。

Q5：AI 產出的內容有著作權問題嗎？

各國法律仍在發展中。實務守則：把 AI 產出當草稿、加入實質人工創作、重要商用內容諮詢法務，並遵守單位的 AI 使用規範與揭露要求。

Q6：學生用 AI 寫作業算作弊嗎？

依各校規範而定。教學上建議引導「AI 當家教與草稿工具、學習成果自己負責」的使用倫理，並要求註明 AI 使用方式。

10.2 十條安全守則（可印成教室海報）

  • 1. 機密與個資不上傳；要用就先去識別化。
  • 2. 事實、數字、法條、醫療資訊——必查證。
  • 3. AI 是副駕駛，最終責任在你。
  • 4. 重要輸出寄出／發布前，人工完整讀一遍。
  • 5. Codex 從保守審批模式開始，熟練再放權。
  • 6. 沒有版本控制（Git）的專案，不要讓代理動手。
  • 7. 代理跑完的程式碼，必看 diff、必跑測試。
  • 8. 提示詞裡不要放帳號密碼與金鑰。
  • 9. 遵守公司與學校的 AI 使用規範，該揭露就揭露。
  • 10. 工具月月更新——訂閱官方公告，教材定期回訓。
